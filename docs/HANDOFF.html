Part 2 — The build (your real job)

P0 The Live Receipts Scoreboard — public, Supabase (deadline: before Sept seal)

The page where our sealed 2026-27 predictions get scored by reality, in public, all season. This is the company's shock-the-world artifact.

PieceSpec
Inputartifacts/sealed_2026.json — schema: {player_id, dest_team, line_no, effective_date, pred_xgf_pct, band_lo, band_hi, achieved_coverage, scenario_hash} + top-level sha256_of_predictions. Produced by scripts/06_seal_predictions.py.
Supabasetables: predictions (sealed rows, immutable), actuals (nightly upserts: gp_so_far, actual_xgf_pct), meta (hash, seal date).
Scoring jobnightly cron: scripts/06_seal_predictions.py --score emits the actuals table (we'll wire a JSON output for you) → push to Supabase.
Pageevery call: player, team, predicted band, live actual, games played, inside-band ✓/✗, running coverage % vs the 80% promise, and the SHA-256 + seal date up top. Design vibe: sports-scoreboard honesty, not marketing gloss. Misses shown as plainly as hits — that IS the brand.

P1 The GM Demo app

Player → destination → line → projection card. Claude provides a local FastAPI endpoint (spec below) wrapping the working projection engine — you own everything above it.

POST /project  {player_id, dest_team, line_no, linemates?:[ids], as_of?:date}
→ {xgf_pct, band:{lo,hi,achieved_coverage,extrapolated:bool},
   env:{xgf60,xga60}, talent:{off,def,n_eff},
   value_lines:{penalty_econ_g60, finishing_g100},
   support:{tier, ok, detail}, scenario_hash}

Card shows: the % + band (with the coverage number — always), the two value lines, the support flag, and dropdowns to swap linemates / line — re-query on change. If extrapolated or REFUSED_OOS: show it loudly. Honesty is the UI's spine.
